Redirecionar as portas

From: https://serverfault.com/questions/112795/how-to-run-a-server-on-port-80-as-a-normal-user-on-linux

Aplicar as regras

Comandos: 
    $ sudo iptables -t nat -A PREROUTING -p tcp --dport 80 -j REDIRECT --to-port 8000 
    $ sudo iptables -t nat -I OUTPUT -p tcp -d 127.0.0.1 --dport 80 -j REDIRECT --to-ports 8000

Remover as regras:

Comando: 
    $ sudo iptables -t nat --line-numbers -n -L 
    $ sudo iptables -t nat -D PREROUTING <num>

Reconfigurar DNS

Editar o arquivo /etc/hosts 
    $ sudo vi /etc/hosts
e incluir o seguinte conteúdo: 
# Para o Play on Linux funcionar
127.0.0.1       repository.playonlinux.com

O diretório resources contém os scripts e os arquivos. Procurar o script POL_ correspondente ao arquivo que deu erro. Remover o endereço original, como no exemplo abaixo: 

        #POL_Download_Resource "https://web.archive.org/web/20190122095451if_/http://download.microsoft.com/download/e/a/f/eafb8ee7-667d-4e30-bb39-4694b5b3006f/msxml6_x86.msi" "85a5571258de322458f288b94ee28cfb"

E colocar no seguinte formato: 

        POL_Download_Resource "http://localhost/download/msxml6_x86.msi" "85a5571258de322458f288b94ee28cfb"

A partir daqui, nada disso é necessário

Criar uma CA

Criar o Diretório para a CA:

    $ sudo mkdir /etc/myCA
    $ cd /etc/myCA

Criar o arquivo de configuração

$ sudo vi openssl.cnf
Cole o seguinte conteúdo no arquivo: 
    [ ca ]
    default_ca = myCA

    [ myCA ]
    dir = /etc/myCA
    new_certs_dir = $dir/newcerts
    unique_subject = no
    certificate = $dir/cacert.pem
    database = $dir/index.txt
    serial = $dir/serial
    private_key = $dir/private/cakey.pem
    default_days = 3650
    default_md = sha256
    policy = myCA_policy
    x509_extensions = myCA_extensions

    [req]
    default_md = sha256
    prompt = no
    distinguished_name = req_distinguished_name
    req_extensions = myCA_extensions

    [req_distinguished_name]
    C = BR
    ST = Rio de Janeiro
    L = Rio de Janeiro
    O = Casa
    OU = Escritorio
    CN = meslin.com.br
    emailAddress = alexandre@meslin.com.br

    [ myCA_policy ]
    commonName = supplied
    stateOrProvinceName = supplied
    countryName = optional
    emailAddress = optional
    organizationName = supplied
    organizationalUnitName = optional

    [ myCA_extensions ]
    basicConstraints = critical, CA:true
    keyUsage = critical, digitalSignature, cRLSign, keyCertSign

Criar Estrutura de Diretórios e Arquivos Iniciais:

    $ sudo mkdir newcerts
    $ sudo touch index.txt
    $ sudo echo '01' | sudo tee serial
    $ sudo mkdir private

Gerar a Chave Privada e o Certificado da CA:

    $ sudo openssl genpkey -algorithm RSA -out private/cakey.pem
    $ sudo openssl req -x509 -new -key private/cakey.pem -out cacert.pem -config openssl.cnf

Criar um certificado para assinar (.csr)

    $ cd /etc/myCA
    $ sudo openssl req -newkey rsa:2048 -keyout key.pem -out cert.csr -nodes -subj "/C=BR/ST=Rio de Janeiro/L=Rio de Janeiro/O=Casa/OU=Casa/CN=meslin.com.br"

Assinar o certificado

    $ cd /etc/myCA
    $ sudo openssl x509 -req -in cert.csr -CA cacert.pem -CAkey private/cakey.pem -CAcreateserial -out cert.pem -days 365 -sha256
Comandos 
    $ sudo apt install python3 python3-pip
    $ sudo pip3 install -r requirements.txt
    $ python3 manage.py runserver_plus --cert-file cert.pem --key-file key.pem
